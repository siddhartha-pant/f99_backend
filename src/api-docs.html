Fitness Tracker API Documentation

This backend is running! Below are the available endpoints, descriptions, and example requests.

POST /api/v1/auth/signup
Register a new user. Returns a JWT token. Automatically creates a profile for the user using the signup information. The user can immediately fetch their profile after signup.

Profile fields not provided at signup (e.g., weight, trainingExperience, aim, bodyMeasurements) will be empty and can be updated later.
Body: { name (required), email (required), password (required), gender (required), age (required) }
Example Request:
{
  "name": "John Doe",
  "email": "john@example.com",
  "password": "secret123",
  "gender": "male",
  "age": 25
}
Example Response:
{
  "message": "Signup successful",
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}
POST /api/v1/auth/signin
Authenticate a user and receive a JWT token.
Body: { email, password }
Example Request:
{
  "email": "john@example.com",
  "password": "secret123"
}
Example Response:
{
  "message": "Signin successful",
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}
POST /api/v1/auth/verify-email
Verify a user's email address using a token (feature currently disabled).
Body: { token }
Example Request:
{
  "token": "your-verification-token"
}
Example Response:
{
  "message": "Email verified successfully"
}
POST /api/v1/profile (JWT required)
Create or overwrite the user's profile.
Body: { weight, age, gender, trainingExperience, aim, bodyMeasurements }
Example Request:
{
  "weight": 70,
  "age": 25,
  "gender": "male",
  "trainingExperience": "beginner",
  "aim": "lose weight",
  "bodyMeasurements": {
    "chest": 90,
    "waist": 80
  }
}
Example Response:
{
  "message": "Profile saved",
  "profile": {
    "_id": "60f7c2...",
    "user": "60f7c2...",
    "weight": 70,
    "age": 25,
    "gender": "male",
    "trainingExperience": "beginner",
    "aim": "lose weight",
    "bodyMeasurements": {
      "chest": 90,
      "waist": 80
    }
  }
}
GET /api/v1/profile (JWT required)
Fetch the user's profile details. All profile fields are always present in the response, with empty values if not set.
Example Response:
{
  "_id": "60f7c2...",
  "user": "60f7c2...",
  "weight": "",
  "age": "",
  "gender": "",
  "trainingExperience": "",
  "aim": "",
  "bodyMeasurements": {}
}
PATCH /api/v1/profile (JWT required)
Update specific fields in the user's profile.
Example Request:
{
  "weight": 72
}
Example Response:
{
  "message": "Profile updated",
  "profile": {
    "_id": "60f7c2...",
    "user": "60f7c2...",
    "weight": 72,
    // ...other fields
  }
}
POST /api/v1/progress (JWT required)
Log daily progress metrics (e.g., weight, energy level, notes).
Body: { weight, energyLevel, notes }
Example Request:
{
  "weight": 71,
  "energyLevel": "high",
  "notes": "Felt great today!"
}
Example Response:
{
  "message": "Progress logged",
  "entry": {
    "_id": "60f7c2...",
    "user": "60f7c2...",
    "weight": 71,
    "energyLevel": "high",
    "notes": "Felt great today!",
    "date": "2025-06-28T10:00:00.000Z"
  }
}
GET /api/v1/progress (JWT required)
Fetch all progress logs for the user.
Example Response:
[
  {
    "_id": "60f7c2...",
    "user": "60f7c2...",
    "weight": 71,
    "energyLevel": "high",
    "notes": "Felt great today!",
    "date": "2025-06-28T10:00:00.000Z"
  }
]
PATCH /api/v1/progress/:entryId (JWT required)
Update a specific daily progress log by its ID.
Example Request:
{
  "notes": "Felt tired today."
}
Example Response:
{
  "message": "Progress updated",
  "entry": {
    "_id": "60f7c2...",
    "user": "60f7c2...",
    "notes": "Felt tired today."
    // ...other fields
  }
}
POST /api/v1/upload/profile-image (JWT required, multipart/form-data)
Upload a profile image for the user. Returns the image URL.
Example Response:
{
  "message": "Profile image uploaded",
  "url": "https://res.cloudinary.com/your-cloud/image/upload/v1234567890/profile.jpg"
}
POST /api/v1/nutrition/meal (JWT required)
Log a meal with calories, macros, and date.
Body: { name, calories, protein, carbs, fat, date }
Example Request:
{
  "name": "Breakfast",
  "calories": 400,
  "protein": 20,
  "carbs": 50,
  "fat": 10,
  "date": "2025-06-15T08:00:00.000Z"
}
Example Response:
{
  "message": "Meal logged",
  "meal": {
    "_id": "60f7c2...",
    "user": "60f7c2...",
    "name": "Breakfast",
    "calories": 400,
    "protein": 20,
    "carbs": 50,
    "fat": 10,
    "date": "2025-06-15T08:00:00.000Z"
  }
}

POST /api/v1/progress-photo/photo (JWT required, multipart/form-data)
Upload a daily progress photo. Optionally include a note. Returns the photo URL and metadata.
Form-data: photo (required, file), note (optional, string)
Example Response:
{
  "message": "Progress photo uploaded",
  "photo": {
    "_id": "60f7c2...",
    "user": "60f7c2...",
    "photoUrl": "https://res.cloudinary.com/your-cloud/image/upload/v1234567890/progress.jpg",
    "note": "Day 1 progress!",
    "date": "2025-06-28T10:00:00.000Z"
  }
}
GET /api/v1/progress-photo/photo (JWT required)
Fetch all uploaded progress photos for the user, sorted by date (most recent first).
Example Response:
[
  {
    "_id": "60f7c2...",
    "user": "60f7c2...",
    "photoUrl": "https://res.cloudinary.com/your-cloud/image/upload/v1234567890/progress.jpg",
    "note": "Day 1 progress!",
    "date": "2025-06-28T10:00:00.000Z"
  }
]

All protected routes require a JWT token in the Authorization: Bearer <token> header.
